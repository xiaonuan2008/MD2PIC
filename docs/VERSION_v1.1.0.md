# MD2PIC v1.1.0 更新日志

**发布日期**: 2026-06-01  
**版本**: v1.1.0

---

## 🎉 本次更新总览

v1.1.0 版本主要修复了P2级问题，并增强了用户体验，为后续功能迭代打下基础。

---

## 🔧 问题修复

### 1. 持久化清空恢复状态 ✅
**问题**: 清空内容后刷新页面，恢复功能失效
**修复**: 
- 将 `lastClearedContent` 持久化到 IndexedDB
- 新增键名 `CLEARED_CONTENT_KEY = 'md_cleared_content'`
- 启动时自动检查是否有可恢复的内容

**文件位置**: `index.html` (相关函数)
- `loadConfigAndDraft()` - 加载时检查恢复状态
- `btnClear.addEventListener()` - 清空时保存状态
- `btnRestore.addEventListener()` - 恢复时清除状态

### 2. 优化图片清理逻辑 ✅
**问题**: 使用 `includes()` 匹配可能误删图片（前缀匹配问题）
**修复**: 
- 使用正则表达式精确匹配 `![...](key)` 格式
- 避免 key1 是 key2 前缀时被误删

**文件位置**: `index.html` 中的 `cleanupUnusedImages()` 函数

### 3. 添加渲染错误边界 ✅
**问题**: 渲染出错时页面白屏，没有友好提示
**修复**: 
- 在 `triggerRender()` 中添加 try-catch
- 出错时显示友好提示和"重置页面"按钮

**文件位置**: `index.html` 中的 `triggerRender()` 函数

### 4. 修复撤销/重做逻辑 ✅
**问题**: 历史记录达到上限时索引处理有问题
**修复**: 
- 优化 `saveHistory()` 函数逻辑
- 当超出50条时，只 shift 不 increment

**文件位置**: `index.html` 中的 `saveHistory()` 函数

---

## ✨ 新增功能

### 1. Toast 提示系统 ✅
**功能描述**: 优雅的消息提示组件

**特性**:
- 支持四种类型：success / error / info / warning
- 自动滑入滑出动画
- 可自定义显示时长（默认3秒）
- 自动清理，避免DOM残留

**使用方式**:
```javascript
showToast('操作成功', 'success');
showToast('发生错误', 'error');
showToast('提示信息', 'info');
showToast('警告信息', 'warning');
```

**应用场景**:
- 清空/恢复操作提示
- 导出成功/失败提示
- 未来更多操作反馈

**文件位置**: 
- CSS: `.toast-container`, `.toast` 相关样式
- JS: `showToast()` 函数

### 2. 导出进度提示 ✅
**功能描述**: 多图导出时显示当前进度

**实现**: 
- 导出按钮文字显示 `正在生成... x/y`
- 完成后显示 Toast 提示

## 🎨 体验优化

### 1. 统一提示风格
- 用 Toast 替换原来的临时 DOM 提示
- 所有用户操作反馈风格一致

### 2. 导出一致性改进
- 使用克隆节点而非移动原节点
- 等待 `document.fonts.ready` 确保字体加载
- 临时容器在视口内但不可见（opacity: 0）
