## 一、🚨 发现的关键问题

### 1️⃣ 严重问题 - 预览与导出不一致（渲染差异）

**问题位置**: [`exportImages()`](file:///workspace/index.html#L1077-L1205) 函数

**问题描述**:  
在导出过程中，为了避免滚动条影响，代码将页面移到视口外：
```javascript
page.style.position = 'absolute';
page.style.left = '-9999px';
page.style.top = '-9999px';
```

**潜在问题**:
1. **字体加载问题**: 当页面移到视口外时，某些浏览器可能会延迟或取消字体加载，导致导出的图片使用默认字体而非用户选择的字体
2. **CSS 伪元素问题**: `::before`、`::after` 等伪元素在绝对定位且脱离文档流时可能出现渲染问题
3. **缩放问题**: 虽然代码重置了 `scaleWrapper.style.transform`，但页面级别的样式可能未完全恢复

**验证建议**:
- 测试使用特殊字体（如站酷字体）导出时是否正常
- 测试包含代码块、数学公式的复杂页面导出

---

### 2️⃣ 严重问题 - 撤销/重做逻辑缺陷

**问题位置**: [`saveHistory()`](file:///workspace/index.html#L564-L574) 函数

**代码**:
```javascript
if (historyStack.length > 50) historyStack.shift(); 
else historyIndex++;
```

**问题描述**:
当历史记录达到 50 条时，执行了 `shift()` 但没有相应调整 `historyIndex`，这会导致：
- 用户撤销时可能出现索引错位
- 历史记录指针指向错误的位置

**影响**: 🟡 中等，日常使用可能没问题，但达到 50 条历史后可能出现异常

---

### 3️⃣ 中等问题 - 图片清理逻辑可能误删

**问题位置**: [`cleanupUnusedImages()`](file:///workspace/index.html#L485-L499)

**问题描述**:
```javascript
if (!currentText.includes(key)) store.delete(key);
```

这个逻辑使用 `includes()` 检查，但如果图片 ID 是 `local_img_123`，而文本中恰好有 `local_img_1234`，也会匹配到，导致误判。

**更安全的做法**: 使用正则表达式匹配完整的图片引用格式 `![...](key)`

---

### 4️⃣ 中等问题 - 缺少渲染错误边界

**问题位置**: [`triggerRender()`](file:///workspace/index.html#L854-L904)

**问题描述**:
该函数没有 `try-catch` 包裹，如果渲染过程中出现任何错误（如 marked 解析失败、图片加载失败等），会导致：
- 页面白屏
- 用户无法继续操作
- 没有任何错误提示

---

### 5️⃣ 中等问题 - 清空与恢复功能的状态管理

**问题位置**: 清空按钮事件处理器 [`btnClear.addEventListener`](file:///workspace/index.html#L701-L730)

**问题描述**:
- `lastClearedContent` 变量是全局变量，但没有持久化到 IndexedDB
- 刷新页面后，清空的内容无法恢复
- 没有检查 `lastClearedContent` 是否为空就显示恢复按钮

---

### 6️⃣ 体验问题 - 导出过程中没有进度反馈

**问题位置**: [`exportImages()`](file:///workspace/index.html#L1077-L1205)

**问题描述**:
- 导出时只有按钮文字变化，没有进度条或百分比
- 导出多页 ZIP 时用户不知道进度
- 可能导致用户重复点击

---

### 7️⃣ 体验问题 - 图片上传没有大小限制

**问题位置**: [`processAndInsertImage()`](file:///workspace/index.html#L780-L800)

**问题描述**:
- 没有限制上传图片的大小
- 超大图片会导致 IndexedDB 存储溢出
- 没有图片压缩功能

## 三、🔧 优先级修复建议

### P0 - 必须立即修复（影响核心功能）

1. **添加渲染错误边界**
   - 在 `triggerRender()` 中添加 try-catch
   - 出错时显示友好提示
   - 提供"重置"按钮

2. **修复预览与导出一致性问题**
   - 方案 1: 使用隐藏的容器而非移到视口外
   - 方案 2: 确保所有样式在导出前完全加载
   - 方案 3: 导出前先克隆节点到新容器

### P1 - 建议尽快修复（影响体验）

3. **修复撤销/重做逻辑**
   ```javascript
   // 修复后
   if (historyStack.length > 50) {
     historyStack.shift();
     // 不要在这里 increment，保持 historyIndex 不变
   } else {
     historyIndex++;
   }
   ```

4. **改进图片清理逻辑**
   - 使用正则表达式精确匹配
   - 检查完整的 `![alt](key)` 格式

### P2 - 可以后续优化

5. **添加导出进度反馈**
6. **添加上传图片大小限制和压缩**
