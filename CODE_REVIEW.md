## 二、潜在 Bug 分析

### 🔴 严重 Bug

#### Bug #1：TOC 功能未实现
**位置**：[index.html#L651-743](file:///workspace/index.html#L651-743)

```javascript
switch (type) {
    case 'bold': /* ... */ break;
    case 'italic': /* ... */ break;
    // ❌ 缺少 'toc' case
    case 'pagebreak':
    case 'hr':
        /* ... */
        break;
}
```

**影响**：点击"📑 目录"按钮只会插入空文本，无任何功能。

**建议修复**：
```javascript
case 'toc':
    beforeText = '\n[[TOC]]\n';
    afterText = '';
    placeholder = '';
    break;
```

并在 `triggerRender()` 中添加 TOC 生成逻辑。

---

#### Bug #2：nextNode 参数未使用
**位置**：[index.html#L1059](file:///workspace/index.html#L1059)

```javascript
function tryAddNode(node, nextNode) {
    // ❌ nextNode 参数声明但从未使用
```

**影响**：代码冗余，可能导致未来逻辑错误。

**建议**：删除未使用的参数，或利用它实现更智能的分页逻辑。

---

### 🟡 中等 Bug

#### Bug #3：IndexedDB 存储无配额管理
**位置**：[index.html#L450-499](file:///workspace/index.html#L450-499)

**问题**：
- 无存储配额限制检测
- 无存储空间不足时的优雅处理
- 大图片累积可能导致存储爆炸

**建议**：
1. 添加存储使用量检测
2. 实现 LRU（最近最少使用）清理策略
3. 添加用户提醒

---

#### Bug #4：导出失败后样式恢复不完整
**位置**：[index.html#L1322-1342](file:///workspace/index.html#L1322-1342)

**问题**：如果 `html2canvas` 在渲染过程中抛出异常，页面的 transform 样式可能无法恢复。

**建议**：在 `finally` 块之外也添加防御性恢复代码。

---

#### Bug #5：移动端切换延迟计算
**位置**：[index.html#L559](file:///workspace/index.html#L559)

```javascript
setTimeout(adjustScale, 50); // ⚠️ 硬编码延迟
```

**问题**：
- 低端设备可能需要更长时间
- 高端设备浪费等待时间
- 50ms 是经验值，不够可靠

**建议**：使用 `requestAnimationFrame` 或 MutationObserver 监听 DOM 变化。
